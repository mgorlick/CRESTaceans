21 Locatives

procedure

(locative/unrestricted/new a) → locative?

  a : actor?
Returns a new unrestricted locative l assigned to actor a, which by default never expires, has infinite amount of sends (messages that can be sent to this actor), has no restrictions on inter- and intra-island sends, it is not revoked nor derived.

procedure

(locative/cons/any l    
  expires    
  sends    
  senders/intra    
  senders/inter) → (or/c #f locative?)
  l : locative?
  expires : (or/c #f (and/c positive? real?))
  sends : (or/c #f (and/c integer? positive?) +inf.0)
  senders/intra : (or/c #t procedure?)
  senders/inter : (or/c #t procedure?)
Permits any actor to create a derivative of any locative l. Given locative l derives and returns a more restricted locative |l'|. Derived locatives form a tree in which a descendant locative grants no more (and often less) capability than its parent locative. The capability of a locative l comprises the expiration date of l, the total number of sends permitted for l, the criteria used to determine which set of intra- and inter- island actors may use l to send a message to the actor guarded by l implemented either by #t (allows any actor to send messages) or a one-argument predicate. The one-argument predicate assigned to senders/intra operates over actors and the one-argument predicate assigned to senders/inter operates over island addresses. The path of l is always prefix of the path of |l'|. The expiration of |l'| is always <= the expiration of l.The total number of sends allocated to |l'| is always <= the total number of sends outstanding for l at the creation time of |l'|. The set of actors that may use |l'| to send a message to the actor denoted by l is always a subset of the set of actors that may use l to send a message to the actor guarded by l.
