2 Type transformations

procedure

(exact->inexact v) → inexact?

  v : number?
Coerces v to an inexact number. If v is already inexact, it is returned.

procedure

(inexact->exact v) → exact?

  v : number?
Coerces v to an inexact number. If v is already exact, it is returned. If v is +inf.0, -inf.0, or +nan.0, then an exception is raised.

procedure

(number->string v [radix]) → string?

  v : number?
  radix : (or/c 2 8 10 16) = 10
Returns a string that is the printed form of v in the base specified by radix. If v is inexact, radix must be 10, otherwise an exception is raised.

procedure

(string->number s [radix]) → (or/c number? #f)

  s : string?
  radix : (integer-in 2 16) = 10
Reads and returns a number datum from s, returning #f if s does not parse exactly as a number datum (with no whitespace). The optional radix argument specifies the default base for the number, which can be overridden by #b, #o, #d, or #x in the string.

procedure

(bytes/list bstr) → (listof byte?)

  bstr : bytes?
Returns a new list of bytes corresponding to the content of bstr. That is, the length of the list is (bytes-length bstr), and the sequence of bytes in bstr is the same sequence in the result list.

procedure

(list/bytes lst) → bytes?

  lst : (listof byte?)
Returns a new mutable byte string whose content is the list of bytes in lst. That is, the length of the byte string is (length lst), and the sequence of bytes in lst is the same sequence in the result byte string.

procedure

(symbol/string sym) → string?

  sym : symbol?
Returns a freshly allocated mutable string whose characters are the same as in sym.

procedure

(string/symbol str) → symbol?

  str : string?
Returns an interned symbol whose characters are the same as in str.

procedure

(vector/list vec) → list?

  vec : vector/persist?
Returns a list with the same length and elements as vec.
