18 Persistent binding environments

syntax

environ/null

Produces a canonical empty binding environment, namely a two element vector vec, where the first element is the symbol ’environ/persist and the second element is a persistent hash table whose keys are the names of lexical variables (given as symbols) and arbirary Motile values.

syntax

BASELINE

The binding environment containing set of primitive procedures available to all Motile mobile code.

procedure

(environ? v) → boolean?

  v : any/c
Returns #t if v is a binding environment, #f otherwise.

procedure

(environ/cons env key v) → environ?

  env : environ?
  key : symbol?
  v : symbol?
Produces a new persistent binding environment by merging the given key/value pair with the given environment env.

procedure

(environ/cons env lst) → environ?

  env : environ?
  lst : list?
Produces a new persistent binding environment from the union of the contents of binding environment env and a list of key/value pairs given as input in the form of (k_0 v_0 ... ... k_i v_i).

procedure

(vector/environ env vec) → environ?

  env : environ?
  vec : vector?
Given a binding environment env and a vector vec = #(k_0 v_0 ... k_ {n-1} v_ {n-1}) where k_i ,v_i is a symbol/value pair, this function returns a successor binding environment containing bindings k_0 v_0 ... k_{n-1} v_{n-1}. Environ env remains unchanged.

procedure

(vectors/environ env keys values) → environ?

  env : environ?
  keys : vector?
  values : vector?
Given a binding environment env and vectors keys = #(k_0 ... k_ {n-1}) corresponding to keys and values = #(v_0 ... v_ {n-1}) corresponding to their respective values, this function returns a successor binding environment containing bindings k_0 v_0 ... k_{n-1} v_{n-1}.Environ env remains unchanged.

procedure

(environ/merge env1 env2) → environ?

  env1 : environ?
  env2 : environ?
Returns the merge of binding environment env2 into binding environment env1 returning a successor binding environment where the bindings of env2 are joined with those of env1. The bindings of env2 take precedence over those of alpha.

procedure

(environ/vector/remove env keys) → environ?

  env : environ?
  keys : vector?
Let keys be a vector (possibly empty) of key names (k_1 ... k_m). This procedure removes the bindings k_1/v_1, ..., k_m/v_m from binding environment env returning a successor environment that does not contain any of k_1/v_1, ..., k_m/v_m. env remains unchanged.

procedure

(environ/ref env key failure) → any/c

  env : environ?
  key : any/c
  failure : any/c
Returns the value for key in binding environment env. key may be a symbol or a vector; otherwise an error will be returned. If no value is found for key, then failure is returned as the result. failure can as well be a procedure that is called in case key is not in env.

procedure

(environ/ref/symbol env key failure) → any/c

  env : environ?
  key : symbol?
  failure : any/c
Returns the value for key in binding environment env. If no value is found for key, then failure is returned as the result. failure can as well be a procedure that is called in case key is not in env.

procedure

(environ/ref/path env keys failure) → any/c

  env : environ?
  keys : vector?
  failure : any/c
Returns the value for key in binding environment env and within a path of its nested binding environments. For example, (environ/ref/path env (vector 'env_sub1 'env_sub2 'hello_world) "failure") is looking for the value of the binding named by symbol 'hello_world within environment env_sub2, which is within environment env_sub1, which exists at the same time within environment env. If no value is found for key, then failure is returned as the result. failure can as well be a procedure that is called in case key is not in env.

procedure

(list/environ env lst) → environ?

  env : environ?
  lst : list?
Given a list lst of bindings (k_1 v_1 ... k_m v_m), join bindings k_1/v_1, ..., k_m/v_m to binding environment env returning the successor environment containing k_1/v_1, ..., k_m/v_m. env remains unchanged.

procedure

(pairs/environ env lst) → environ?

  env : environ?
  lst : (listof pair?)
Given a list of pairs of bindings lst ((k_0 . v_0) ... (k_i . v_i)), join bindings k_0/v_0 ... k_i/v_i to binding environment env returning the successor environment containing k_0/v_0 ... k_i/v_i. env remains unchanged.
