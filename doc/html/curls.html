22 Capability URL (CURL)

procedure

(curl/new/any l path meta [lifespan]) → (or/c #f curl?)

  l : locative?
  path : (listof symbol?)
  meta : (or/c #f (and/c hash/persist? hash/eq?))
  lifespan : list? = null
Creates a new unsigend CURL derived from locative l on the following conditions: locative l must not be expired nor revoked; the lifespan of the new CURL (expressed in seconds and included as the first element of list lifespan) must be equal to or smaller that the expiration of locative l; and locative/sends/positive? must not be #f, meaning that there are still sends allowed to this locative. Otherwise it returns #f.

The null path is equivalent the path name /. The path (s_1 s_2 ... s_n) is equivalent to the path name /s_1/s_2/.../s_n.

procedure

(curl/new l path meta [lifespan]) → (or/c #f curl?)

  l : locative?
  path : (listof symbol?)
  meta : (or/c #f (and/c hash/persist? hash/eq?))
  lifespan : list? = null
Equivalent to curl/new/any, but imposes the additional restriction of checking whether the actor creating the curl is in the curl/new whitelist of locative l, namely if the result of locative/curl/authority? is #t.

procedure

(curl/id c) → (or/c locative? pair? symbol?)

  c : curl?
Returns either the id of the locative CURL c represents or the locative itself. It this is an inter-island CURL it will return the locative’s id represented by a pair or symbol. If instead the CURL is intra-island, it will return the locative.

procedure

(curl/target=? c1 c2) → boolean?

  c1 : curl?
  c2 : curl?
Returns #t if CURLs c1 and c1 share both the same island address and the locative they represent. This means that the results of (curl/id c) and (curl/island c) must be the same for both.

procedure

(curl/send c msg) → boolean?

  c : curl?
  msg : any/c
Using CURL c as a destination address, sends message msg to the actor denoted by CURL c. Returns #t on successful delivery and #f otherwise. If the sender desires a reply, then the CURL metadata should contain an item reply/x where reply is the symbol reply and x is the CURL to which the reply should be directed.

procedure

(curl/send/multiple cl msg) → boolean?

  cl : (listof curl?)
  msg : vector?
Using the CURL within the list cl as a destination addresses, sends message msg to all the actors denoted by those CURLs. Returns #t on successful delivery and #f otherwise. If the sender desires a reply, then the CURL metadata should contain an item reply/x where reply is the symbol reply and x is the CURL to which the reply should be directed.
